# Git understøttet processer

den gitunderstøttede produktforvaltning vil skulle understøtte flg processer

1. Indberetning, evaluering og prioritering af ændringsønsker
2. Indberetning, evaluering og prioritering af fejl og mangler
3. Diskusion og dokumentation af arkitekturprincipper, produktstrategi og samarbejdsmodel
4. Risiko styring
5. Leverancesledelse og fremdriftsrapportering
6. backlog evaluering og releasestyring
7. Intern Opgavestyringog opfølgning


Den gitunderstøttede produtktforvaltning forudsætter at:
- produktudviklingen foregår i produkt organisationens git repositores og styres via git issues
- produktets vedligeholdelses leverandører vedligeholder produktets git repositories (maintainer rolle)
- Der anvendes Semanttisk versionering og conventional commits
- Der anvendes standardiseret navngivning og labeling af Issues
